# Деплой Goals Tracker на Amvera Cloud — подробная инструкция

## Что такое Amvera и зачем всё это

Amvera — это российский облачный хостинг. Ты загружаешь свой код через git (как на GitHub), и Amvera сам собирает и запускает твоё приложение.

У нас в проекте три части, и для каждой нужен **отдельный проект** в Amvera:

1. **PostgreSQL** — база данных (хранит все цели)
2. **Backend** — Express API-сервер (принимает запросы от фронтенда, читает/пишет в БД)
3. **Frontend** — React SPA (то, что видит пользователь в браузере)

Создавать нужно **именно в таком порядке**, потому что каждый следующий зависит от предыдущего.

---

## ЭТАП 1. Регистрация на Amvera

1. Открой браузер и перейди на **https://cloud.amvera.ru/**
2. Нажми **«Зарегистрироваться»**
3. Заполни форму: email, логин, пароль
4. Подтверди email (придёт письмо)
5. Войди в аккаунт

> **Важно:** запомни свой **логин** (username). Он будет использоваться в URL-адресах. Например, если твой логин `oleg`, то адреса будут содержать `oleg`.

## ЭТАП 2. Создать базу данных PostgreSQL

Нашему backend нужна база данных, чтобы хранить цели. Создадим её первой.

### 2.1. Открой страницу создания

1. В личном кабинете Amvera (https://cloud.amvera.ru/) нажми кнопку **«Создать проект»**
2. Выбери тип сервиса: **«СУБД»**
3. Выбери **«PostgreSQL»**

### 2.2. Заполни параметры

- **Название проекта**: `goals-db` (или любое другое, латинскими буквами, без пробелов)
- **Тариф**: выбери **«Начальный»** (минимальный рабочий тариф для PostgreSQL)

На следующем шаге появятся поля для настройки самой базы:

- **Имя создаваемой БД**: `goals_tracker`
- **Имя пользователя**: `goalsuser` (нельзя использовать слово `postgres` — оно зарезервировано)
- **Пароль пользователя**: придумай надёжный пароль, например `MyStr0ngP@ss2026` — **запиши его, он понадобится дальше!**
- **Размер кластера СУБД**: `1` (одной реплики достаточно)
- **Активировать Superuser Access**: не обязательно, можно не ставить галочку

### 2.3. Завершить создание

1. Нажми **«Завершить»**
2. Подожди пока статус не сменится на **«PostgreSQL запущен»** (обычно 1-3 минуты)

### 2.4. Скопировать адрес подключения

Это самый важный момент — нам нужно узнать «адрес» базы данных внутри облака Amvera.

1. Открой проект `goals-db` в Amvera
2. Перейди на вкладку **«Инфо»**
3. Найди строку **«Внутреннее доменное имя для чтения/записи»** — она будет выглядеть примерно так:
   ```
   amvera-oleg-cnpg-goals-db-rw
   ```
   (вместо `oleg` будет твой логин, вместо `goals-db` — название проекта)
4. **Скопируй это имя** — оно понадобится на следующем этапе

### 2.5. Собрать строку подключения

Теперь собери полную строку подключения (DATABASE_URL) из данных выше. Формат:

```
postgres://ИМЯ_ПОЛЬЗОВАТЕЛЯ:ПАРОЛЬ@ВНУТРЕННИЙ_ХОСТ:5432/ИМЯ_БАЗЫ
```

Пример (подставь свои значения):

```
postgres://goalsuser:MyStr0ngP@ss2026@amvera-oleg-cnpg-goals-db-rw:5432/goals_tracker
```

**Запиши эту строку** — она нужна для backend.

## ЭТАП 3. Создать проект для Backend

### 3.1. Создать проект в Amvera

1. В личном кабинете нажми **«Создать проект»**
2. Тип сервиса: **«Приложение»**
3. Заполни:
   - **Название**: `goals-backend`
   - **Тариф**: **«Начальный»**
4. Нажми **«Далее»**

### 3.2. Пропустить загрузку данных

На шаге «Загрузка данных» — **ничего не загружай**. Мы будем загружать код через git из терминала. Просто нажми **«Далее»**.

### 3.3. Добавить переменные окружения

Это очень важный шаг. Переменные окружения — это настройки, которые твоё приложение читает при запуске. Мы не храним пароли в коде — мы передаём их через переменные.

На шаге «Переменные» (или потом в разделе «Переменные» проекта) добавь **4 переменных**. Для каждой нажми **«Добавить переменную или секрет»**:

**Переменная 1:**
- Тип: **Секрет** (потому что содержит пароль)
- Название: `DATABASE_URL`
- Значение: строка подключения из этапа 2.5, например:
  `postgres://goalsuser:MyStr0ngP@ss2026@amvera-oleg-cnpg-goals-db-rw:5432/goals_tracker`

**Переменная 2:**
- Тип: Переменная
- Название: `NODE_ENV`
- Значение: `production`

**Переменная 3:**
- Тип: Переменная
- Название: `PORT`
- Значение: `4000`

**Переменная 4:**
- Тип: Переменная
- Название: `FRONTEND_ORIGIN`
- Значение: пока напиши `https://example.com` — мы изменим это позже, когда создадим frontend

### 3.4. Пропустить конфигурацию

На шаге «Конфигурация» — **ничего не выбирай, не заполняй**. У нас уже есть файл `amvera.yml` в коде. Если задать конфигурацию через интерфейс, она может конфликтовать.

Нажми **«Завершить»**.

### 3.5. Загрузить код backend через git

Теперь самое интересное — нужно отправить код backend в Amvera. Поскольку наш проект — это один репозиторий с двумя папками (`backend/` и `frontend/`), а Amvera ожидает, что `amvera.yml` лежит в корне, мы используем специальную команду `git subtree push`. Она отправляет **только содержимое одной папки** как отдельный репозиторий.

Открой терминал, перейди в корень проекта и выполни команды **по одной**:

```bash
# 1. Перейди в папку проекта (если ещё не там)
cd /Users/oleg/Documents/vibe-coder/target-app

# 2. Добавь удалённый репозиторий Amvera для backend.
#    URL бери со страницы «Репозиторий» проекта goals-backend в Amvera.
#    Формат: https://git.amvera.ru/ТВОЙ_ЛОГИН/goals-backend
git remote add amvera-backend https://git.amvera.ru/ТВОЙ_ЛОГИН/goals-backend

# 3. Отправь ТОЛЬКО папку backend в Amvera.
#    Эта команда берёт содержимое папки backend/ и пушит его
#    как будто это корень отдельного репозитория.
git subtree push --prefix=backend amvera-backend master
```

> **При выполнении команды push** Amvera попросит ввести логин и пароль — введи логин и пароль от **аккаунта Amvera** (не от GitHub!).

> **Если появилась ошибка** `fatal: refusing to merge unrelated histories` — это значит, что Amvera создал конфигурацию через интерфейс. Выполни:
> ```bash
> git push amvera-backend `git subtree split --prefix=backend`:master --force
> ```

### 3.6. Дождаться сборки

1. Вернись в личный кабинет Amvera
2. Открой проект `goals-backend`
3. Статус сменится на **«Выполняется сборка»** — подожди
4. Если всё хорошо, статус станет **«Работает»** (зелёный)
5. Если статус **«Ошибка сборки»** или **«Ошибка при запуске»** — перейди в раздел **«Логи»** и посмотри, что пошло не так

### 3.7. Подключить домен

Чтобы backend был доступен по HTTPS-ссылке:

1. Открой проект `goals-backend` в Amvera
2. Перейди в раздел **«Настройки»**
3. Найди блок **«Доменное имя»**
4. Нажми **«Активировать бесплатное доменное имя Амвера»**
5. Подожди пока привязка завершится (1-5 минут)
6. Появится адрес вроде: `goals-backend-oleg.amvera.tech`
7. **Скопируй этот адрес — он нужен для frontend**

### 3.8. Проверить что backend работает

Открой в браузере:

```
https://goals-backend-ТВОЙ_ЛОГИН.amvera.tech/api/goals?year=2026&month=4
```

Если всё хорошо, ты увидишь:

```json
{"data":[]}
```

Это значит: backend работает, подключился к базе, таблица целей пока пустая. Отлично!

Если видишь ошибку — проверь логи в Amvera и переменные окружения (особенно `DATABASE_URL`).

## ЭТАП 4. Создать проект для Frontend

### 4.1. Создать файл с адресом backend

Фронтенд должен знать, куда отправлять API-запросы. Этот адрес «вшивается» при сборке. Поэтому нужно создать файл `.env.production` **до** отправки кода в Amvera.

Открой терминал в корне проекта и выполни:

```bash
# Перейди в корень проекта
cd /Users/oleg/Documents/vibe-coder/target-app

# Создай файл .env.production для фронтенда.
# ЗАМЕНИ goals-backend-oleg.amvera.tech на ТВОЙ реальный домен из шага 3.7!
echo "VITE_API_BASE_URL=https://goals-backend-ТВОЙ_ЛОГИН.amvera.tech/api" > frontend/.env.production
```

Убедись, что файл создался правильно:

```bash
cat frontend/.env.production
```

Должно вывести что-то вроде:

```
VITE_API_BASE_URL=https://goals-backend-oleg.amvera.tech/api
```

Теперь закоммить это изменение:

```bash
git add frontend/.env.production
git commit -m "Add frontend production env with Amvera backend URL"
```

### 4.2. Создать проект в Amvera

1. В личном кабинете нажми **«Создать проект»** → **«Приложение»**
2. Заполни:
   - **Название**: `goals-frontend`
   - **Тариф**: **«Пробный»** или **«Начальный»** (фронтенд — это просто статические файлы, ему хватит минимума)
3. **Пропусти** загрузку данных (нажми «Далее»)
4. **Пропусти** переменные (фронтенду они не нужны — всё уже в `.env.production`)
5. **Пропусти** конфигурацию (у нас есть `amvera.yml`)
6. Нажми **«Завершить»**

### 4.3. Загрузить код frontend через git

```bash
# 1. Перейди в корень проекта
cd /Users/oleg/Documents/vibe-coder/target-app

# 2. Добавь remote для frontend
#    URL бери со страницы «Репозиторий» проекта goals-frontend в Amvera
git remote add amvera-frontend https://git.amvera.ru/ТВОЙ_ЛОГИН/goals-frontend

# 3. Отправь ТОЛЬКО папку frontend в Amvera
git subtree push --prefix=frontend amvera-frontend master
```

Введи логин и пароль Amvera при запросе.

### 4.4. Дождаться сборки

1. Открой проект `goals-frontend` в Amvera
2. Дождись статуса **«Работает»**

### 4.5. Подключить домен

1. Перейди в **«Настройки»** проекта `goals-frontend`
2. Активируй **бесплатное доменное имя**
3. Подожди привязки
4. Появится адрес вроде: `goals-frontend-oleg.amvera.tech`

## ЭТАП 5. Настроить CORS (разрешить фронтенду обращаться к backend)

Backend по умолчанию блокирует запросы с чужих доменов (это называется CORS — Cross-Origin Resource Sharing). Нужно сказать ему, что запросы с домена фронтенда — разрешены.

1. Открой проект **`goals-backend`** в Amvera
2. Перейди в раздел **«Переменные»**
3. Найди переменную `FRONTEND_ORIGIN` и **измени** её значение на домен фронтенда:
   ```
   https://goals-frontend-ТВОЙ_ЛОГИН.amvera.tech
   ```
   > Важно: именно `https://`, без `/` на конце!
4. Нажми **«Сохранить»**
5. **Перезапусти** контейнер backend: на странице проекта нажми кнопку **«Перезапустить»** (или «Restart»)

---

## ЭТАП 6. Проверить что всё работает

1. Открой в браузере: `https://goals-frontend-ТВОЙ_ЛОГИН.amvera.tech`
2. Должна загрузиться таблица целей
3. Попробуй добавить цель — если она сохраняется и показывается после перезагрузки страницы, значит всё работает!

## Как обновлять проект после изменений

Когда ты поменял код локально и хочешь обновить на Amvera:

```bash
# 1. Закоммить все изменения
git add -A
git commit -m "описание что изменил"

# 2. Запушь в GitHub (чтобы код был и там)
git push origin main

# 3. Обнови backend на Amvera (если менял файлы в backend/)
git subtree push --prefix=backend amvera-backend master

# 4. Обнови frontend на Amvera (если менял файлы в frontend/)
git subtree push --prefix=frontend amvera-frontend master
```

После каждого push в Amvera автоматически запустится пересборка.

## Частые проблемы и их решения

### «Ошибка сборки» в Amvera
- Перейди в раздел **«Логи»** проекта — там будет написано, что именно сломалось
- Чаще всего: забыли закоммитить `package-lock.json` или `amvera.yml`

### Backend не подключается к базе
- Проверь что `DATABASE_URL` в переменных указан правильно
- Формат: `postgres://user:password@host:5432/dbname`
- Host — это **внутреннее доменное имя** со страницы «Инфо» PostgreSQL проекта

### Фронтенд открывается, но данные не загружаются
- Открой DevTools в браузере (F12) → вкладка Console — там будет ошибка
- Если ошибка CORS — проверь `FRONTEND_ORIGIN` в backend (шаг 5)
- Если ошибка сети (Network Error) — проверь что `frontend/.env.production` содержит правильный URL бэкенда

### `git subtree push` выдаёт ошибку
- Если ошибка `Updates were rejected` — значит репозитории рассинхронизировались. Выполни:
  ```bash
  git push amvera-backend `git subtree split --prefix=backend`:master --force
  ```
  (замени `backend` на `frontend` для фронтенда)
